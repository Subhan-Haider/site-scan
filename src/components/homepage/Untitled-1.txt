        <h3>Supported Image Formats</h3>
        <ul>
          <li><strong>PNG</strong> — Portable Network Graphics. Lossless compression with full transparency (alpha channel) support. Ideal for logos, screenshots, and illustrations.</li>
          <li><strong>JPG / JPEG</strong> — Joint Photographic Experts Group. Widely supported lossy compression format optimal for photographs and complex imagery.</li>
          <li><strong>WEBP</strong> — Modern image format developed by Google offering 25–34% better compression than JPEG at equivalent visual quality. Fully supported in all modern browsers.</li>
          <li><strong>AVIF</strong> — AV1 Image File Format. Next-generation codec with up to 50% better compression than JPEG and 20% better than WEBP. Excellent for high-DPI displays.</li>
          <li><strong>SVG</strong> — Scalable Vector Graphics. XML-based format for resolution-independent icons, logos, and illustrations. Infinitely scalable without quality loss.</li>
          <li><strong>ICO</strong> — Windows Icon format. Multi-resolution container format for browser favicons and application icons.</li>
          <li><strong>BMP</strong> — Bitmap Image File. Uncompressed raster format developed by Microsoft. Maximum color fidelity with large file sizes.</li>
          <li><strong>TIFF</strong> — Tagged Image File Format. High-fidelity multi-page container format widely used in professional printing, scanning, and archiving workflows.</li>
          <li><strong>PDF</strong> — Portable Document Format. Universal document standard for text, images, and vector graphics — fully editable and annotatable within ImgConvert Pro.</li>
        </ul>

        <h3>Tool 5: Grayscale B&amp;W Converter</h3>
        <p><strong>Technical Overview:</strong> This tool utilizes a GPU-accelerated canvas filter pipeline to remove color. First, pdf.js renders each page into an offscreen canvas. We retrieve the canvas's 2D context image data as a raw 32-bit RGBA pixel buffer. For every pixel, the luminance value is calculated using the standard CCIR 601 formula: <code>L = 0.299R + 0.587G + 0.114B</code>. The red, green, and blue values are replaced with this calculated luminance, converting the image to true grayscale. The modified pixel buffer is written back to the canvas, and a new high-resolution PDF is compiled with the black-and-white canvas layers.</p>
        <p><strong>Step-by-Step Guide:</strong></p>
        <ol>
          <li>Open the <strong>Grayscale &amp; B&amp;W</strong> utility in the PDF tools interface.</li>
          <li>Upload your colored PDF document.</li>
          <li>Adjust the threshold slider to adjust black-and-white contrast levels.</li>
          <li>Select your target DPI (150 DPI for standard use, 300 DPI for high-resolution print).</li>
          <li>Click "Convert to Grayscale". The client-side worker processes each page canvas sequentially.</li>
          <li>Once finished, download your print-ready black-and-white PDF.</li>
        </ol>

        <h2>Modern Browser Sandboxing &amp; Client-Side Media Engineering</h2>
        <p>ImgConvert Pro uses a local-first architecture to provide desktop-class media processing performance directly in your browser. Here is an overview of the core technologies that make this possible:</p>

        <h3>1. WebAssembly (WASM) Sandbox Compilation</h3>
        <p>For complex image compression tasks (like AVIF encoding) and character recognition (OCR), standard JavaScript can be slow. ImgConvert Pro bypasses these performance limits by compiling high-performance C++ and Rust libraries into WebAssembly (WASM) binaries. These binary files run at near-native speeds inside the browser's sandbox environment, enabling fast, complex data processing without relying on external cloud servers.</p>

        <h3>2. Offscreen Canvas &amp; Multi-threaded Web Workers</h3>
        <p>Processing hundreds of images at once can freeze a website's user interface. To prevent this, ImgConvert Pro offloads heavy processing tasks to background threads using browser <strong>Web Workers</strong>. Combined with <strong>OffscreenCanvas</strong> APIs, these workers perform image scaling, color conversions, and canvas filtering in the background, keeping the user interface smooth and responsive throughout.</p>

        <h3>3. Active Browser Memory Management &amp; Security</h3>
        <p>Processing files locally requires careful memory management, as loading massive image batches or large PDFs can quickly exceed browser limits. ImgConvert Pro uses a dynamic memory allocation pipeline that processes files sequentially or in small batches. Once a file is processed, we release its memory immediately using the browser's <code>URL.revokeObjectURL()</code> API, avoiding memory leaks and keeping resource usage low. Furthermore, because all data is stored in temporary browser memory, it is permanently deleted as soon as you close the tab, keeping your files completely secure.</p>
